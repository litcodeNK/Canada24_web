STEP 5 — NETWORK CONNECTIVITY ISSUE & FIX
================================================================================

PROBLEM:
  Device network layout:
    ap0 (hotspot) → 10.160.133.51   (phone acting as WiFi AP)
    Mac IP        → 10.160.133.14   (Mac connected to phone hotspot)
    ccmni1        → 10.126.166.155  (phone on cellular internet)

  Django was started on 127.0.0.1:8000 → device could not reach it.
  Restarted Django on 0.0.0.0:8000 → macOS firewall blocked port 8000
  on the hotspot interface (Python.framework binary not whitelisted).

  Test: adb shell wget http://10.160.133.14:8000/health/ → FAIL

FIX — ADB Reverse Port Tunneling (bypasses firewall via USB):
  adb reverse tcp:8000 tcp:8000
  adb reverse tcp:8081 tcp:8081

  This forwards device's localhost:8000 → Mac's localhost:8000
  and device's localhost:8081 → Mac's localhost:8081 (Metro bundler)

  .env updated:
    BEFORE: EXPO_PUBLIC_API_BASE_URL=http://10.160.133.14:8000/api/v1
    AFTER:  EXPO_PUBLIC_API_BASE_URL=http://127.0.0.1:8000/api/v1

  Metro bundler restarted to pick up new .env value.
  App reloaded on device.

  Test: curl http://127.0.0.1:8000/health/ → {"status":"ok",...} ✅

NOTES FOR NEXT SESSION
----------------------
1. Keep .env as http://127.0.0.1:8000/api/v1 when using USB/ADB dev setup.
2. Re-run ADB reverse tunnels after each USB reconnect:
     adb reverse tcp:8000 tcp:8000
     adb reverse tcp:8081 tcp:8081
3. Start backend with:
     cd backend
     DJANGO_SETTINGS_MODULE=config.settings.development python3 manage.py runserver 0.0.0.0:8000
4. Start Metro with:
     npx expo start --port 8081
5. To switch back to hotspot mode (no USB), change .env back to the Mac's
   hotspot IP and allow Python through macOS firewall:
     sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add \
       "/Library/Frameworks/Python.framework/Versions/3.12/Resources/Python.app/Contents/MacOS/Python"
     sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblockapp \
       "/Library/Frameworks/Python.framework/Versions/3.12/Resources/Python.app/Contents/MacOS/Python"
